# Post-Launch Review Template

## Review Schedule
- 24h review
- 7d review
- 30d review

## 24h Review
### Stability
- Availability (%)
- Error rate (5xx + client errors)
- Incidents and rollback actions

### Performance
- P95 API latency
- P95 page load time
- Slowest endpoints

### Product Flows
- Register/Login success rate
- Checkout success rate
- Order update flow health

## 7d Review
### Trends
- Error trend vs launch day
- Traffic trend vs forecast
- Resource utilization trend (CPU/RAM/DB)

### Operational Learnings
- Alerts quality (signal vs noise)
- Runbook effectiveness
- Team response time and gaps

## 30d Review
### Outcome Summary
- SLA/SLO adherence
- Major incidents and fixes
- Top bottlenecks removed

### Technical Debt / Next Actions
- Reliability improvements backlog
- Performance optimization backlog
- Monitoring and alert tuning backlog

## Lessons Learned
- What worked well
- What failed or was unclear
- What to change in next release process

## Sign-Off
- QA lead
- DevOps lead
- Backend lead
- Frontend lead
- Product owner
